학습내용

<1일차>
Atom 이라는 Editor를 다운받고, 가장 기본적인 태그 (strong, u, h1 ~ 6, br 등을 배웠다.) 줄바꿈 Tag에는 br 과 p가 있다. br은 단순히 줄바꿈이며, p는 단락(paragraph)를 나타낸다.
p는 해당 문단의 처음과 끝에 사용하면 되고, br은 줄 바꾸고 싶은 곳마다 사용하면 된다.
image를 page에 삽입시킬 때 사용하는 tag는 img이다.
src = source라는 뜻. 'img src="" ' 라고 작성하면 이미지 사용이 가능하다.
img src="test image 1.jpg" width="100%"로 작성한 이미지가 위와 같다. 이때, src와 width의 위치는 서로 바뀌어도 된다.

<2일차>
목차는 'li'라는 태그를 사용한다. 만약 2개로 나누어진 목차를 사용하고싶다면, li 태그의 '부모 태그'인 ul을 사용하면 된다.
하지만 이러한 목차에는 한 가지 큰 문제점이 발생한다. 그것은 바로 '목차를 수정할 때'이다. 위의 목차는 현재 수기로 1,2 ... 숫자를 넣었다. 만약, 이 목차에 추가하거나 삭제해야 할 내용이 있을 경우, 우리는 직접 숫자를 바꾸어주어야 한다. 이런 경우, 'ol'태그를 사용하면 해결할 수 있다.
ul = unordered list / ol = ordered list
본문은 'body'라는 태그로 묶기로 약속했고, 본문(body)를 설명하는 태그는 'head'태그로 묶기로 약속했다. 이 두 개의 태그를 묶는 'html' 태그를 사용한다.
'title'태그가 없다면, 검색 엔진은 존재할 수 없다.
링크를 걸고 싶은 곳에 'a' 태그를 사용한다. 그 후, 해당 링크에대한 정보를 달아준다. (a href="link"). 해당 링크를 눌렀을 때, 새 탭에서 열리게 하고싶으면 target="_blank"를 사용하고, 툴팁을 추가할 경우, title 태그를 추가해주면 된다.

<3일차>
3일차에는 각각의 목록에 링크를 걸어주었다. 방법은 2일차에 배웠던 'a' 태그를 이용하는 것이다. 2일차 수업의 복습이라고 볼 수 있다.
웹은 1990년에 처음 나오게 되었다. 인터넷과 웹은 어떻게 생겨나게 되었는가 ? 인터넷과 웹은 서로 다르다. 인터넷이 도시라면, 웹은 그 도시위에 있는 건물 하나이다. 혹은, 인터넷이 도로라면, 웹은 도로 위의 한 대의 차라고 할 수 있다. 인터넷은 1960년에 등장했다. 1960년 당시, 인류는 핵 전쟁을 치룬 직후였다. 따라서, 사람들은 핵에 대한 두려움이 컸다. 미국에서는 만약, 핵 공격을 당했을 경우 어떻게 되는가를 시뮬레이션을 해보았다. 그 결과, 통신 시스템이 중앙집중적이었기 때문에 통신이 마비되는 심각한 상황이 발생했다. 이러한 핵 공격에도 버틸 수 있는 통신망으로 개발된 것이 바로 인터넷이다. 개발 초기, 인터넷은 기업이나 연구소 등 거대한 기관에서만 사용했기 때문에 일반 사람들은 인터넷을 잘 몰랐다. 하지만, 1990년 웹을 만나게 되면서 사람들에게 널리 알려지게 된다.

<4일차>
웹 브라우저가 설치되어있는 컴퓨터는 웹 서버가 설치되어있는 컴퓨터에 정보를 '요청'하며, 반대의 상황에선 '응답'한다. 이 때, 웹 브라우저 = 클라이언트(Client), 웹 서버 = 서버(Server)라고 부른다.
